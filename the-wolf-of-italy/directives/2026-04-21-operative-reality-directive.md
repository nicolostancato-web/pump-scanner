# DIRETTIVA FONDATORI — The Wolf of Italy
**Emessa il 2026-04-21 — Priorità assoluta**

## PRINCIPIO FONDAMENTALE
Tutto deve essere vero. Nei report può comparire solo ciò che esiste davvero.

Un team è attivo solo se ha flusso reale, input reale, output reale, memoria reale.

## CLASSIFICAZIONE TEAM (obbligatoria nei report)
- A. OPERATIVO REALE: flusso + task + output + note + tracking reali
- B. IN COSTRUZIONE: funzione chiara, workflow in build
- C. SOLO PIANIFICATO: definito sulla carta, non ancora costruito

## POLITICA ZERO COSTI
Tutto ciò che è gratuito, utile e legale: il CEO lo attiva subito.
Approvazione fondatori solo per: nuova spesa, abbonamento, API key da fornire, investimento, rischio legale/reputazionale.

## FLUSSI REALI MINIMI DA COSTRUIRE
1. CEO Report Flow — raccolta stato reale + report mattina/sera
2. Research Flow — task → esecuzione → note → scheda opportunità
3. Execution Flow — opportunità → test → risultato → learning
4. CFO Flow — costi attuali + nuovi costi + costi evitati + alert
5. QC Flow — verifica notes + output + qualità
6. Security Flow — audit credenziali + accessi + esposizioni

  output_summary.md

## REGOLA FINALE
Tutto deve essere vero. Tutto deve essere tracciabile. Se gratis e utile: si fa subito. Se una pista può fare soldi: si prova subito.
